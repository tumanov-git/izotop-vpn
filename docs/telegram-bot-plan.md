# Izotop Connect Telegram Bot Plan

Дата: 2026-03-26

## Цель

Собрать рабочий Telegram-бот для `Izotop Connect`, который:

- ведёт пользователя по понятному интерфейсу через `inline keyboards`;
- показывает статус доступа;
- выдаёт `subscription URL` из `Remnawave`;
- показывает инструкции для нужного устройства;
- принимает `Tribute webhooks` и обновляет статус подписки;
- даёт админке быстрый ручной контроль над пользователями.

## Принятые решения

### Инфраструктура

- `Remnawave Panel` живёт на польском `management VPS`
- `Remnawave Node` живёт на старом NL-сервере
- клиентский `REALITY` на NL идёт через `8443/tcp`
- связь `Panel -> Node` идёт через `2222/tcp`

### Домены

- `panel.izotop-connect.online`
- `sub.izotop-connect.online`
- `nl-001.izotop-connect.online`

### Доступ пользователя

- один пользователь `Tribute` = один пользователь `Remnawave`
- один пользователь `Remnawave` = один `subscription URL`
- одна подписка используется на нескольких устройствах
- тип устройства влияет на инструкцию, а не на генерацию отдельного VPN-ключа

### Tribute

- источник обновлений — `webhooks`, не постоянные API-проверки
- используем:
  - `new_subscription`
  - `renewed_subscription`
  - `cancelled_subscription`
- уже существующих старых пользователей можно занести вручную в админке

## Пользовательский путь

### Пользователь с активной подпиской

1. Нажимает `/start`
2. Видит статус подписки
3. Выбирает `Получить доступ` или `Мои ключи`
4. Выбирает устройство
5. Получает:
   - краткую инструкцию
   - `subscription URL`
   - QR-код

### Пользователь без подписки

1. Нажимает `/start`
2. Видит, что подписка не найдена или истекла
3. Может:
   - оформить доступ
   - повторно проверить статус
   - написать в поддержку

## Админка в боте

Только для администраторов.

### Задачи админки

- искать пользователя по `telegram_user_id`
- смотреть текущий статус
- вручную заносить старых подписчиков
- вручную продлевать доступ
- создавать или пересоздавать `subscription URL`
- видеть историю обработанных webhook-событий

## Интеграции

### Tribute webhook flow

1. Прилетает webhook
2. Проверяется подпись
3. Событие дедуплицируется
4. Обновляется подписка в БД
5. Если нужно, создаётся или актуализируется VPN-доступ

### Remnawave flow

1. Пользователь нажимает `Получить доступ`
2. Бот ищет `vpn_account`
3. Если нет — создаёт пользователя в `Remnawave`
4. Если есть — использует сохранённый `subscription URL`
5. Бот отдаёт ссылку и QR
